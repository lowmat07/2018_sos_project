proposal

For my Student Originated Software programming project, I propose to design and implement a new model of information display on the internet. I will create a template for an entry in an expressions thesaurus as the model.

The idea for the template is to allow a user to add new sayings, add new classifications to existing sayings, add alternative grammar spins of existing sayings, etc. The challenge is to create a system of data storage and linking between miscellaneous elements in a way that primarily reuses rather than repeats stored information.

I intend on making the model in 2 parts following the client-server model that is used by most websites. I will start on the client-side first to produce a demo of what it could look like for the user. I will next work on server side storage of reuired data, and then will produce a first client/server prototype.

client side

  • create a working concept prototype
  • interactive functionality is more important than design as the ultimate goal is to have multiple interfaces for different of users
  • a JSON virtual database can be used to simulate database access

server side

  • find database system with capacity for large storage
  • choose a database structure that easily permits the fields and contents of storage

extras

  • integrate git storage for site
  • make demonstration site responsive
